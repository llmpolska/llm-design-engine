# LLM Design Engine v0.1 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a private, clean-checkout MVP that compiles product meaning into variable scene graphs, validates provider output, applies semantic refinement, exposes MCP tools, and proves the workflow with real screenshots and cross-domain fixtures.

**Architecture:** Keep `packages/core` as the canonical Zod/type contract owner. Extend `packages/creative-director` with an explicit meaning-to-composition compiler and provider response validator, then let CLI, MCP, Studio, renderer, benchmark, screenshots, and website consume the same `AgentExecutableDesign` contract. Keep demo examples in fixtures and make all production mock output algorithmic from brief-derived signals.

**Tech Stack:** TypeScript strict mode, Zod, Turborepo, pnpm workspaces, Vue 3/Vite, Hono, Playwright, Vitest, `@modelcontextprotocol/sdk`, `zod-to-json-schema`, Changesets, GitHub Actions.
